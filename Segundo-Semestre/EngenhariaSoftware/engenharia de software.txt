PROCESSO DE SOFTWARE

é um conjunto de atividades, ações e tarefas

ATIVIDADES: esforça-se para atingir um objetivo amplo(ex.: modelagem)

AÇÕES: (ex.: projeto de arquitetura) conjunto de tarefas que resultam num subproduto

TAREFAS: (ex.: elaborar diagrma de classes) se concentra em um objetivo pequeno e 
bem definido e produz um resultado tangivel




MODELO CASCATA: util quando os requisitos de um problema são compreendidos

comunicação: inicio do projeto, levantamento de necessidades

planejamento: estimativas, cronograma, acompanhamento

modelagem: análise, projeto

construção: codificação, teste

emprego: entrega, suporte, feedback


PROTOTIPAÇÃO
é um modelo de desenvolvimento de software no qual o protótipo é construído,
testado e retrabalhado até que um protótipo aceitável seja alcançado. 
Também cria base para produzir o sistema ou software final.


MODELO ESPIRAL
é um dos mais importantes modelos de ciclo de vida de desenvolvimento de 
software, que fornece suporte para gerenciamento de riscos.


ENGENHARIA DE SOFTWARE
é a área responsável pela criação, manutenção e desenvolvimento 
de sistemas
